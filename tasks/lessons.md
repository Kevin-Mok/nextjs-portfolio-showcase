# Lessons

- 2026-03-13: When adding homepage CTAs or navigation affordances in this repo, preserve the transparent/glass visual language by default. Do not introduce filled backgrounds in the tiled content area unless the user explicitly asks for them.
- 2026-03-28: When tailoring a resume variant from source documents, confirm the exact experience mix the user wants before locking in multiple roles from the same employer. Do not assume both cloud and support entries should stay just because both are source-backed.
- 2026-03-28: When adding or refreshing resume project entries, verify the public repo/demo URL from source docs or explicit user input. Do not leave a project unlinked when the canonical public repo is known.
- 2026-03-28: When a resume screenshot shows obvious underfill, do not rely on one-page status or minimum whitespace floors alone. Add a regression for the requested section order/content and enforce a tighter per-variant bottom-whitespace ceiling when the user asks for fuller page fill.
- 2026-03-30: When asked to add resume content for a new project, do not replace an existing project direction with a different one unless the user explicitly confirms that swap. Preserve the current project entry by default and treat replacement as a separate decision.
- 2026-03-30: When mentioning approximate line-number amounts or file line-count magnitudes, round up to the nearest hundred instead of citing exact counts.
- 2026-03-30: When mentioning approximate line-number amounts or file line-count magnitudes, use coarse rounded figures: below 1000, round up to the nearest hundred; at 1000 or above, round down to the nearest multiple of 500 and add a `+`.
- 2026-04-10: When a user asks to remove a specific resume bullet, delete that bullet literally. Do not replace it with adjacent tooling copy unless the user explicitly asks for a rewrite.
